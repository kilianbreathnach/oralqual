Image Modelling with Astrometry.net

with David Hogg and Dustin Lang (CMU)

Astrometry.net

  • Compares sources found in image to a catalogue with "geometric hashing"
    • Super fast and ZERO false positives
  • Can be used to astrometrically calibrate any astronomical image (optical)
  • Crowdsourcing: currently ~50k pictures tagged #astrometry on Flickr

But...

  • Calibration only achieves accurate positions of astronomical sources
  • We would also like to know about source brightnesses - enables more science
  • Calibration done using an imperfect catalogue (USNO-B)

Idea

  • Construct Image model with an image catalogue of stars, galaxies, background... Prior knowledge about model from the output of Astrometry.net code
  • Use model to generate a fake data image and compare with real data
  • Improve the comparison by including missing stars, motion of stars, supernovae... Science!
  • Do this for many images to build up a probabilistic catalogue for the sky

The HOD: What is it and why bother?

  • \( \langle N_{central}\rangle_{M} = \frac{1}{2} \left[1 + erf\left(\frac{log(M) - log(M_{min})}{\sigma_{logM}} \right) \right] \)
  • \( \langle N_{satellites}\rangle_{M} = \left[\frac{M - M_{cut}}{M_{sat}}\right]^{\alpha_{sat}} \)
  • Used to populate N-body dark matter simulations with galaxies, it succesfully reproduces observable clustering statistics as seen in galaxy surveys! A successful method to account for non-linear galaxy bias, cool!

Void Probability Function

  • Probability that a randomly placed sphere in space, of a given radius, contains no galaxies
  • e.g. for a Poisson distribution of points, \( P_{0} (r) = P(k = 0, r) = \frac{\lambda^{k}}{k!} e^{-\lambda r} = e^{-\lambda r} \)
